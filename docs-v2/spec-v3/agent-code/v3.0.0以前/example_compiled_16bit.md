# example.c の16bit Synthetica Scriptコンパイル結果

## このファイルについて

- `example.c` のC言語コードを16bit Synthetica Scriptへコンパイルした結果
- 16bitアーキテクチャ仕様に基づく実装
- 最新のユニット操作メモリ領域仕様に準拠
- **更新**: 1024進法エネルギーシステムに対応（2025-07-28）

## コンパイル仕様

- 16bitアーキテクチャ（プログラムカウンタ、レジスタ、メモリアドレスすべて16bit）
- 変数はメモリへ格納（16bit値）
- ユニットインデックスはDレジスタ経由で動的指定
- テンプレートは`template_XX`形式で表現（XXは8bit値の16進数表現）

## メモリマップ

```
0x0000-0x01FF: プログラムコード領域
0x0200-0x02FF: 変数領域
  0x0200: child_hull_index (16bit)
  0x0202: memory_size (16bit)
  0x0204: memory_address (16bit)
  0x0206: temp_value (16bit) - 一時変数
0x0300-0x03FF: スタック領域（未使用）
0x0400-0xFFFF: 予約領域
```

```
# エネルギー関連定数（1024進法システム）

CHILD_HULL_TOTAL_COST       = 0x00D2  # 210E（200E + 10E生産コスト）
CHILD_ASSEMBLER_TOTAL_COST  = 0x04B0  # 1200E（800+200 + 200生産コスト）
```

## 主な変更点（現在の仕様への対応）

1. **HULL操作**
   - 容量読み取り: 0x00アドレス
   - マージ: 0x04アドレスで相互指定方式
   - 分離: 0x05-0x07アドレス

2. **ASSEMBLER操作**
   - 生産状態: 0x08アドレス（True=生産中、False=完了/待機）
   - 最後の生産情報: 0x0C-0x0Dアドレス
   - リセット: 0x0Eアドレス

3. **COMPUTER操作**
   - メモリアドレス指定: 0x01アドレス
   - メモリ値読み書き: 0x02アドレス
   - 外部書き換え許可: 0x00アドレス（デフォルトTrue）

## エネルギーコスト概算

### 命令実行コスト（新エネルギーシステム）

- 1バイト命令: 1E
- 3バイト命令: 3E
- 4バイト命令: 4E（ユニット操作は+10E）
- 5バイト命令: 5E

### プログラム概要

- プログラムサイズ: 約512バイト (0x0200)
- 成長ループ: 約100命令（約300E/ループ）
- 複製ループ: 約150命令（約450E/ループ）
- メモリコピー: 256バイト×3E = 768E

### 自己複製に必要なエネルギー合計

- 娘個体生成: 16,045E（210E + 1200E + 14635E）
- 作業エネルギー: 約1,500E（命令実行 + メモリコピー）
- **合計**: 約17,545E = 17×1024E + 137E（1024進法表現）

```assembly
JL insufficient_energy         # エネルギー不足なら分岐

# エネルギー消費の例
# 消費エネルギーがC,Dレジスタに格納済み
SUB_E32                        # A,B = A,B - C,D（エネルギー減算）

# 概算計算の例（kE単位）
SHR_E10                        # A = 上位16bit（1024E単位の値）
```

### 利用可能なエネルギー計算命令

- **0x95: ADD_E32** - 32bitエネルギー加算
- **0x96: SUB_E32** - 32bitエネルギー減算
- **0x97: CMP_E32** - 32bitエネルギー比較
- **0x98: SHR_E10** - 1024で除算（上位取得）
- **0x99: SHL_E10** - 1024倍（左シフト）

## 注意事項

- ユニットインデックスは固定値を仮定（HULL=0, ASSEMBLER=0, 新規COMPUTER=1）
- エラー処理は最小限（生成失敗時のdetachのみ）
- マージ操作は両HULLでの相互指定が必要
- 外部COMPUTERへの書き込みは許可状態を前提

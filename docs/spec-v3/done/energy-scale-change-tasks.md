- 上位16bit: 1024E単位（2^10）
- 下位16bit: 1E単位

## 1024進法の利点

- **ビット演算での効率的な実装**
  - 下位10bitマスク: `lower & 0x3FF`
  - 上位へのキャリー: `upper += (lower >> 10)`
  - 1024での除算: 10bit右シフト
  - 1024倍: 10bit左シフト

- **メモリ効率**
  - 最大値: 65,535 × 1024 + 1,023 = 67,108,863E
  - 十分な範囲をカバー
